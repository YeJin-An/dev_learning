type conversion
-> 하나의 타입을 다른 타입으로 바꾸는 행위를
   타입 변환(type conversion)

// 주의점 -> 피연산자들을 모두 같은 타입으로 만든 후 수행

-> 표현 범위가 좁은 타입 -> 표현 범위가 큰 타입 경우
: 타입 변환은 큰 문제가 되지 않는다.
-> 표현 범위가 큰 타입 -> 표현 좋은 타입 경우
: 데이터의 손실이 발생한다.


1. 묵시적 타입 변환(자동 타입 변환, 
                    implicit type conversion)

-> C컴파일러가 자동으로 실행 변환 _산술,대입
-> 산술에서 데이터손실을 최소화하는 묵시적 타입변환
// 데이터를 저장하는데, 데이터 손실을 최소하기 위한 1차변환


2. 명시적 타입 변환(강제 타입 변환,
                    explicit type conversion)

-> 사용자가 type cast연산자를 사용하여
   강제적으로 수행하는 타입 변환을 가리킨다.
-> ("()") 이것을 타입 캐스트(type cast)라고 한다.
ex> double result02 = (double)num01 / num02;
    결과값으로 double데이터 타입의 데이터를 출력한다.

// 정리
// 피연산자의 타입과 언제나 일치하기 때문에,
// 하나의 피연산자를 명시적으로 double형으로 지정해야만
  정확한 결괏값을 얻을 수 있습니다.

-> 피연산자는 어셈블리어 언어의 
데이터 타입과 피연산자(Operand)타입이 있습니다.
