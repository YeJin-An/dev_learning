1. 마일스톤(프로젝트 관리)__체크
: 프로젝트 진행 과정에서 특정할 만한 건이나 표
EX> 프로젝트 계약, 작수, 인력두입, 선금 수형, 중간보고, 감리, 종료, 잔금 수령
    프로젝트 성공을 위해 반드시 거쳐야 하는 지점

: 프로젝트 일정관리를 위해 체크하는 용도로 사용 한다.
: 전체적인 일정지연없이 제 사간안에 과업이 종료될 수 있도록 도움을 준다.
: 중요한 핵심적인 사항들만 체크하기 때문에, {상세한 내용은 [파악]}하기 힘들다.




2. PERT(Program Evaluation and Review Technique)__프로젝트 평가 검토 기봅
: 팀의 업무 효율설을 향상하고자 하는 프로젝트 관리자에게 매우 유용.

RERT기법 of PERT의 역사
-> 프로젝트의 전반적인 평가로 시각하는 기법
-> 프로젝트 개시하기 전 프로젝트에 대한 심츠적인 분석이 가능하게 한다.

RERT 프로젝트 관리의 원리
-> 순자적인 작업 or 동시실행을 통해 완료해야하는 프로젝트에 사용하도록 설계됨.
프로젝트 목표
: 프로젝트의 전체 범위를 정확하게 평각하고, 예산내에서 완료하는 것이다.

RERT 프로젝트의 범위 설정
-> 프로제그 실행 계획을 수립한다__ '주요 일정'을 수립하는 기법이다.
-> 자원관리에서 개별적인 성능까지 살펴보기 때문에 지나치게 세심해 보일 수 있음.

RERT 범위에 관한 용어
: 중요한 이벤트와 시기, 자원을 정의한다.
1. 단계(Event) : 프로세스에서 이정표 역할
2. 활동(Activity) : 자원[시간,자제,비용,기계 등]을 필요로 하는 작업
3. 시간(Time) : 핵심적인 요소, 최적화된 성능과 프로세스 강의 차질 감안
   -> 낙관치, 비관치, 기대시간, 평소대로 진행되면, 중간정도가 될 가능성이 높다.
4. 관리 : 각 단계나 활동의 기대 시간과 중요성, 진행 상황을 정의
   -> 임계 경로(Critical Path), 임계 활동(Critical Activity), 
      여유(Float OR Slack), 선행 시간(Lead Time), 지체 시간(Lag Time)
      공정 중첩 단축법(Fast Tracking), 공정 압축법(Crashing)

PERT 프로젝트 계획의 단계
1. 작업 파악
2. 적절한 순서 정하기
3. 시간 예측
4. PERT 도표 작성
5. 여유 할당

PERT를 활용한 작업 계획
-> PERT 표를 작성하면 [기간],[활동 순서] 정하는 데, 도움이 됨.
: RERT 네트워크 공정표 만들기
-> 프로젝트의 순서를 시각적으로 일려주는 순서도의 역할
-> 온라인에서 간트 차트로 이용할 수 있음 [낙관치, 비관치, 기대시간 포함해야함.]

PERT 기법의 장단점 : 계획 단계애서조차도 상당한 노력이 요구된다
-> PERT 기법의 장점 7가지존재 , 단점 6가지 존재

PERT 기법이 적합한 프로젝트__PERT 네트워크 공정표를 정사진으로 활용
-> 많은 인력과 작업이 수반되는 프로젝트에서는 
   강력한 프로젝트 관리 도구를 제공하는 것과 유사하다
-> 복잡하지 않는 프로젝트는 프로젝트 분석하느라 시간낭비와 혼란만 가중 시킬 수 있음.




CPM (Critical Path Method)
: 프로젝트 완료에 필요한 태스크를 식별하고 예약 유연성을 결정하는 기술
: 프로젝트 타임라인 
  -> 1. 중요한 작업을 발견  2. 작업 종속성을 파악  3. 작업 기간을 계산하는 것을 중심
: 1950년대, 비효율적인 일정 수립으로 인한 비용 증가 문제를 해결하기 위한 방법
  -> 프로젝트 기획과 과제 우선순위 지정이 대중화되었다.
  -> 프로젝트를 개별작업으로 유연성 보다 잘 이해할 수 있도록 지원.
