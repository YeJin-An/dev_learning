작업분류체계(Breakdown Structure)
: 프로젝트를 완료하는 데 필요한 작업의 계층적 개요.
: 프로젝트 구조를 관리 가능한 결과물로 '분류'
: 프로젝트의 요구 사항을 충족하기 위해 세분화하는 일련의 작업
-> 프로젝트 수명 주기 관리의 일부이다.

WBS의 목적 -> 프로젝트 일정을 계획하는 것
           -> 작업 기간은 [필수선행 작업] 및 [후속 작업]과 함께 계획
           -> 프로젝트 관리자가 워크플로를 관리할 수 있게 전체 계획 제공

WBS 구성 요소
-> 작업 : 제목 및 설명
-> 작업 소유자 : 작업 완료를 담당하는 사람
-> 작업 종속서 및 선행 작업 : 중첩된 업무를 연결
-> 작업 시작 및 완료 날짜 : 각 작업 및 전체에 소용되는 시간 추정
-> 기간 : 각 작업이 소용되는 시간
-> 작업 견적 : 작업 완료까지 필요한 작업 시간/일
-> 작업 상태 : 각 작업이 쇼유자.리소스에 할당되었는지 여부
-> Gantt 차트 : 시간 경과에 따라 시각화 작업.

WBS__프로젝트 비즈니스 자동화 청사진
-> 나머지 프로젝트 프로세스 및 시스템에 통합하는 방법
-> 프로젝트 운영과 재무를 연결하고 프로젝트 비즈니스를 위한
   포괄적인 비즈니스 시스쳄을 만들기 위한 최종 가이드

WBS와 CBS의 차이점
-> WBS는 어떤 작업이 완료되고,
         어떤 일정으로 완료될 것인지에 대한 운영 계획

-> CBS는 프로젝트의 재무 계획
         비용을 효율적으로 관리할 수 있을 정도로 세분화되어 있음.

// CBS는 재정적 목적으로 프로젝트를 분류할 수 있고, 
         효과적인 [프로젝트 비용 관리] 및 [획득 가치 분석]이 가능합니다.
