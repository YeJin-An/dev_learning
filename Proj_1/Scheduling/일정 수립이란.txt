일정계획 수립

일정(Schedule) 개념
-> 프로젝트의 프로세스를 이루는 소작업을 파악 및 예측한다.
-> 프로젝트의 프로세스를 이루는 소작업의 순서와 일정을 정한다.

일정 수립 원칙
1. 분할
-> 프로젝트는 관리 가능한 작업들이 분할되어야 한다.

2. 상호 의존__관계
-> 분할된 작업들 간에 어떤 관계와 상호 의존성이 결정되어야 한다.

3. 시간 할당
-> 각 작업에 시간을 할당해야 한다.

4. 노력 확인__올바르게 할당
-> 소프트웨어에 팀원들이 시간이 올바르게 할당되었는지 확인해야 한다.

5. 책임성__특정 팀에게 할당
-> 계획된 작업이 특정 팀에게 할당되어야 한다.

6. 정의된 산출물.이정표__결과물
-> 각 작업들은 꼭 산출물과 이정표를 가지고 있어야 한다.



일정(Schedule) 수립 방법
1. 막대차트__간트차트[Gannt chart], 시간선(Time-Line)차트
-> 작업의 시작과 종료를 알리는 프로젝트 일정표
-> 가로 막대로 활동기간과 일정벙보를 보여준다.
// 약 8가지의 장점이 존재한다.

2. 마일스톤 차트
-> 막대차트와 유사하지만, 예정시작일 or 예정종료일 AND 외부 인터페이스만 보여준다.

3. PERT(Program Evaluation and Review Technique) / CPM(Critical Path Method)
: 네트워크는 프로젝트의 지연을 방지하고, 계획대로 진행되도록 일정이 계획하는 것
: 조직적인 추진을 위한 자원축소와 단시간 계획 완성을 위한 프로젝트 일정 방법
(1) 특징
(2) 임계경로 (Critical Path)
-> 제품을 개발(완성)되기 까지 가장 많은 시간을 소요하는 경로

(3) PERT(Program Evaluation and Review Technique)
-> 낙관 시간치(optimistic estimate time:a)
   : 순조롭게 진행될 때의 최단 소요시간.
-> 정상 시간치(most likely estimate time:m)
   : 정상적인 조건에서 보편적으로 소용되는 활동 시간.
-> 비관 시간치(pessimistic estimate time:b)
   : 가장 불리한 상황이 전개될 때의 최장 소요시간

(4) CPM(Critical Path Method)
: 프로젝트 완성에 필요한 작업을 나열 및 소요 시간을 예측하는 데 사용하는 기법
-> 활동 중심의 확정적 모델로 목표기일 단축과 비용 최소화를 달성하기 위해 작성한다.
: 노드와 간선으로 구성된 네트워크
-> 간선은 작업 사이의 전후 의존 관계를 나타난다.
: 반드시 이전의 작업이 모두 완료되어야 다른 이정표에 도달가능하다.
: 각 작업이 수행되는 시간과 각 작업 사이의 관계를 파악가능
